- Thêm `--` trước pattern để ngăn pattern bắt đầu bằng `-` bị hiểu là flag.
- Fix `RelativePath` trong kết quả: trước đây gán sai `psi.Arguments`, nay tính đúng bằng cách bỏ prefix root.
- Xem fix logging bên dưới để debug tiếp nếu vẫn không có kết quả.

- [x]: Bổ sung thêm log lệnh tìm kiếm khi thực hiện tìm kiếm trong output window, log rõ ràng lệnh chạy, thời gian thực thi, và kết quả output của Ripgrep

**Triển khai (`ProjectHelperViewModel.cs`):**
- Log đầy đủ trước khi chạy: ripgrep path, thư mục tìm, lệnh đầy đủ với args
- Log khi thử direct execution vs cmd.exe fallback
- Log exit code và số kết quả parse được
- **Log toàn bộ stderr** của ripgrep — đây là nguồn thông tin quan trọng để debug (ví dụ: "rg: path not found", "No such file", permission errors)

- [x]: Chức năng làm tươi trong task history panel chưa hoạt động đúng

**Nguyên nhân:**
1. `HistoryQueryService.GetCacheAsync()` trả về cache cũ nếu `_cacheDirty == false`. `FileSystemWatcher` chỉ set dirty khi file được tạo/xóa, nhưng không đủ reliable trong mọi tình huống.
2. Nút `🔄` trong HistoryControl gọi `_vm.RefreshAsync()` không có flag force, nên cache không bị invalidate.

**Fix:**
- `HistoryQueryService`: thêm method `InvalidateCache()` — set `_cacheDirty = true` và clear `_cache`.
- `FileSystemWatcher`: thêm `_watcher.Changed` event để bắt thay đổi nội dung file (ngoài Created/Deleted).
- `HistoryViewModel`: thêm `_invalidateCache` action (injected từ package), thêm `forceInvalidate` param vào `RefreshAsync()`.
- `HistoryControl.BtnRefresh_Click`: gọi `_vm.RefreshAsync(forceInvalidate: true)`.
- `DevTaskTrackerPackage`: wire `() => _historyRepo.InvalidateCache()` vào HistoryViewModel constructor.

- [x]: Checksum bị báo "thay đổi" mặc dù file chưa bị chỉnh sửa từ bên ngoài

**Nguyên nhân gốc:** Có sự không nhất quán giữa cách tính checksum khi archive và cách verify khi load:
- **Archive** (`JsonStorageService.ArchiveReportAsync`): serialize `CompletionReport` thành JSON (Checksum = null), rồi dùng `JsonConvert.SerializeObject` lần 2 — chuỗi JSON có thể chứa `"Checksum": null`.
- **Verify** (`HistoryQueryService.VerifyReportChecksum`): parse raw JSON từ file (có `"Checksum": "sha256hash"`), remove key "Checksum", `obj.ToString(Formatting.Indented)` — chuỗi này KHÁC với chuỗi archive ban đầu nếu archive có `"Checksum": null`.

**Fix:** Cả hai phía đều dùng cùng một approach: serialize → JObject.Parse → Remove("Checksum") → ToString(Formatting.Indented).

Cụ thể trong `ArchiveReportAsync`:
```
1. JsonConvert.SerializeObject(report)  → tempJson (có "Checksum": null hoặc không có)
2. JObject.Parse(tempJson)
3. jobj.Remove("Checksum")
4. jobj.ToString(Formatting.Indented)  → canonicalJson  ← HASH CỦA CÁI NÀY
5. SetChecksum(hash)
6. JsonConvert.SerializeObject(report)  → finalJson (có Checksum đúng)
7. WriteAllText(finalJson)
```

Bây giờ verify cũng làm đúng y chang (parse file → remove Checksum → ToString → hash). 

- [x]: Phát triển thêm tính năng TODO List

**a) TodoTemplates — chọn từ danh sách, button Add enabled:**

**Vấn đề:** Nhấn template button không enable nút Add vì `AddTodoCommand.CanExecuteChanged` không được raise khi `NewTodoText` thay đổi.

**Fix (`TrackerViewModel.cs`):**
- `NewTodoText` setter giờ gọi `(AddTodoCommand as RelayCommand)?.RaiseCanExecuteChanged()` sau khi set.
- `AddTodoFromTemplateCommand` đổi behavior: thay vì auto-add, **chỉ set `NewTodoText = t`** — nút Add được enable, user xem/chỉnh rồi mới click Add. Điều này cho phép user review trước khi add.

**b) TODO buttons không đúng trạng thái (start/pause/stop không disable sau khi dùng):**

**Nguyên nhân:** `RaiseStateChanged()` trong `TodoItemViewModel` chỉ gọi `OnPropertyChanged` cho các property, nhưng WPF không tự động re-evaluate `ICommand.CanExecute` trừ khi `CanExecuteChanged` được raise.

**Fix (`TodoItemViewModel.cs`):** Trong `RaiseStateChanged()`, bổ sung gọi `RaiseCanExecuteChanged()` cho tất cả 4 commands:
```csharp
(StartCommand    as RelayCommand)?.RaiseCanExecuteChanged();
(PauseCommand    as RelayCommand)?.RaiseCanExecuteChanged();
(StopCommand     as RelayCommand)?.RaiseCanExecuteChanged();
(CompleteCommand as RelayCommand)?.RaiseCanExecuteChanged();
```

Kết quả: Button Start disable sau khi click, Pause chỉ enable khi đang Running, Stop chỉ enable khi Running hoặc Paused — đúng state machine.
