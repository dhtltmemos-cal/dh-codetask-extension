### Thực hiện tiếp tục:

- [x] TodoTemplates — chọn từ danh sách, button Add enabled (chưa triển khai đúng)

**Nguyên nhân 1 — Expander không hiển thị:**
`BooleanToVisibilityConverter` yêu cầu input là `bool`, nhưng `TrackerControl.xaml` bind `Visibility` vào `TodoTemplates.Count` (kiểu `int`). WPF silent-fail → Expander luôn `Collapsed`. User không nhìn thấy template nào dù đã cấu hình trong settings.

**Fix:** Thêm property `HasTodoTemplates { get => TodoTemplates.Count > 0; }` vào `TrackerViewModel`, bind Expander vào `HasTodoTemplates`. Khi `RefreshTodoTemplates()` chạy, gọi thêm `OnProp(nameof(HasTodoTemplates))`.

**Nguyên nhân 2 — Template button không enable nút Add:**
(Đã fix trong v3.7 — `NewTodoText` setter raise `AddTodoCommand.CanExecuteChanged`)

- [x] TODO con chỉ chạy được khi task cha đang Running (hiện task đang ngưng nhưng TODO vẫn start được)

**Nguyên nhân:** `TodoItemViewModel.CanStart` chỉ kiểm tra state của chính item (`Idle` hoặc `Paused`), không kiểm tra state của task cha. Khi task cha ở trạng thái Paused/Stopped, TODO vẫn có thể nhấn ▶.

**Fix:**
- `TodoItemViewModel`: thêm `Func<bool> _isParentRunning` constructor param.
  ```
  public bool CanStart =>
      (Model.Status == Idle || Model.Status == Paused)
      && (_isParentRunning?.Invoke() ?? true);
  ```
- Thêm `RefreshParentStateCommands()` method — gọi `OnPropertyChanged(nameof(CanStart))` và `StartCommand.RaiseCanExecuteChanged()`.

- `TrackerViewModel.CreateTodoVm()`: truyền `isParentRunning: () => TimerState == "Running"`.

- `TrackerViewModel.TimerState` setter: gọi `RefreshTodoParentStateCommands()` sau khi TimerState đổi.
  ```csharp
  private void RefreshTodoParentStateCommands()
  {
      foreach (var vm in Todos)
          vm.RefreshParentStateCommands();
  }
  ```

**Kết quả:**
- Task đang `Running` → nút ▶ trên TODO item enabled (nếu item chưa done)
- Task `Paused` / `Stopped` / `Idle` → nút ▶ trên TODO item disabled
- Khi task Resume → nút ▶ tự enable lại ngay lập tức
